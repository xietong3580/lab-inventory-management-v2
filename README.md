# 库存自动化管理系统 V2

一个基于 **React + Vite + Tailwind CSS** 构建的库存自动化管理系统前端原型项目，面向实验室耗材与库存管理场景。

本项目是对旧版库存系统的 **独立重做 V2 版本**，目标不是直接改造现有线上系统，而是在不影响旧系统使用的前提下，重新搭建一个结构更清晰、视觉更统一、便于后续扩展真实业务逻辑的新版前端基础。

---

## 项目定位

本项目的核心目标包括：

- 重设计库存系统 UI
- 优化后台布局与信息层级
- 提升系统的专业感、规范感与品牌感
- 以 mock 数据完成前端业务原型搭建
- 为后续接入真实后端、数据库与业务逻辑打下清晰基础

项目当前重点不是“快速接后端”，而是先把前端业务骨架、交互流程和多页面联动能力做扎实。

## 当前已实现能力

### 产品与库存管理
- **产品管理闭环**：新增、编辑、删除产品，支持搜索、分类筛选与分页联动
- **库存状态自动计算**：根据 `currentStock` 与 `minStock` 自动推导库存状态（正常/低库存），无需人工维护
- **低库存预警联动**：Alerts 页面动态展示低库存产品，Dashboard 预警数量自动更新
- **多页面数据同步**：Products、Dashboard、Alerts 基于统一数据源，库存变化后各页面自动刷新

### 交易与库存联动
- **出入库记录管理**：新增入库/出库交易，出库时自动进行库存不足校验
- **交易驱动库存变化**：每笔交易自动影响产品库存数量，库存变化自动触发状态重新计算
- **交易撤销与回滚**：支持撤销已完成交易，自动执行库存回滚并标记已撤销状态
- **最近交易记录展示**：Dashboard 实时展示最近出入库记录与状态

### 审计日志与操作追溯
- **审计日志底层系统**：自动记录产品增删改、出入库、交易撤销、系统重置等关键操作
- **Dashboard 最近操作记录**：首页展示最近 5 条操作日志，快速感知系统动态
- **AuditLog 独立页面**：完整日志列表查看，按时间倒序展示所有操作记录
- **基础筛选能力**：支持按操作类型筛选、按产品名称关键词搜索，提升日志查找效率

### 系统维护与开发调试
- **localStorage 本地持久化**：产品、交易、日志数据刷新后不丢失，适合单机演示与测试
- **Settings 重置本地测试数据**：提供安全可控的 UI 入口，支持一键重置所有本地数据到初始状态
- **SYSTEM_RESET 审计记录**：系统重置操作自动记录审计日志，保持操作可追溯性

## 项目结构

以下为当前项目核心结构示意：

```bash
src/
├── pages/
│   ├── Products.jsx
│   ├── Transactions.jsx
│   ├── Dashboard.jsx
│   ├── Alerts.jsx
│   ├── Users.jsx
│   ├── Settings.jsx
│   └── Login.jsx
├── services/
│   └── productService.js
├── constants/
│   └── mockData.js
├── layouts/
│   └── MainLayout.jsx
├── components/
│   └── ...
└── App.jsx
```
结构说明
pages/：页面级模块
services/productService.js：产品、库存、交易记录等核心业务逻辑
constants/mockData.js：初始 mock 数据
layouts/：主布局结构
components/：通用组件与页面复用组件

可按实际代码结构继续调整。

技术栈
React
Vite
Tailwind CSS
项目特点
独立于旧库存系统，不影响现有系统使用
风格简洁、专业、克制，偏企业后台
使用 mock 数据，便于前端结构和 UI 演示
避免营销官网风格和明显 AI 拼装感
适合作为后续真实业务开发的前端基础
采用分阶段迭代方式推进，便于持续扩展与维护

## 当前阶段边界

当前项目仍采用 **前端内存 mock 数据** 方案，因此：

- 页面刷新后，新增记录和库存变化会恢复为初始 mock 数据

- 尚未实现权限管理、审批流、批量导入导出等企业级功能

这些属于当前阶段设计边界，不影响项目作为业务原型的展示价值。

## 下一步方向（第五阶段建议方向）

第四阶段已完成高级组合筛选、CSV 导出能力、多页面体验统一与界面细节优化，系统已具备现代企业后台的完整业务演示能力、高级查询导出能力与基础多端可用性。后续第五阶段可考虑以下方向：

- **查询方案管理**：保存常用筛选组合、查询历史记录、查询方案复制与分享

- **模板自定义**：用户自定义导出模板、字段选择、格式设置、样式配置
- **批量操作**：批量导出多个页面数据、定时导出、导出任务队列与导出历史

### 方案 D：用户体验深度优化与后端接入准备
- **性能优化**：大数据量虚拟滚动、懒加载、缓存策略、首屏加载优化与渲染性能提升
- **移动端体验**：移动端细节优化、触摸手势支持、离线模式、PWA 支持与网络状态处理
- **无障碍支持**：键盘导航、屏幕阅读器支持、高对比度模式、语音控制与可访问性优化
- **后端接入准备**：API 接口设计、数据模型完善、认证授权方案、前后端分离架构部署

截至目前，本项目已经从静态前端页面演进为具备完整业务闭环、历史追溯能力与专业报表界面的现代企业后台系统，已实现：

- **完整业务闭环**：产品管理、出入库记录、库存计算、交易撤销、审计追溯
- **多时间维度分析**：今日、近7天、近30天、全部四种时间范围的筛选与数据分析
- **专业报表界面**：Dashboard KPI卡片、趋势汇总、风险卡片、零值优化、模块平衡
- **历史追溯能力**：产品库存变动历史查看（MVP），基于交易与日志数据整合
- **响应式基础适配**：小屏导航抽屉化、表格页横向滚动策略、关键页面移动端展示策略
- **企业级界面规范**：统一的视觉风格、信息密度优化、专业克制的设计表达
- **高级查询导出能力**：多维度组合筛选、自定义日期范围、CSV导出、体验统一
- **系统维护工具**：本地数据持久化、测试数据重置、全局返回顶部等辅助功能
- **多页面数据联动**：Products、Transactions、Dashboard、Alerts、Settings、AuditLog 六页面数据同步

项目已具备现代企业后台的完整业务演示能力、高级查询导出能力与专业界面基础，为后续深化数据可视化、高级报表分析及后端接入提供了清晰的前端架构。
