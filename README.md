# 库存自动化管理系统 V2

一个基于 **React + Vite + Tailwind CSS** 构建的库存自动化管理系统前端原型项目，面向实验室耗材与库存管理场景。

本项目是对旧版库存系统的 **独立重做 V2 版本**，目标不是直接改造现有线上系统，而是在不影响旧系统使用的前提下，重新搭建一个结构更清晰、视觉更统一、便于后续扩展真实业务逻辑的新版前端基础。

---

## 项目定位

本项目的核心目标包括：

- 重设计库存系统 UI
- 优化后台布局与信息层级
- 提升系统的专业感、规范感与品牌感
- 以 mock 数据完成前端业务原型搭建
- 为后续接入真实后端、数据库与业务逻辑打下清晰基础

项目当前重点不是“快速接后端”，而是先把前端业务骨架、交互流程和多页面联动能力做扎实。

## 当前已实现能力

### 1. 产品管理模块
- 产品列表展示
- 产品搜索
- 分类筛选
- 分页联动
- 新增产品
- 编辑产品
- 删除产品

### 2. 库存状态自动计算
项目中不再手动维护库存状态，而是根据业务规则自动推导：

- 当 `currentStock <= minStock` 时，自动标记为 **低库存**
- 当 `currentStock > minStock` 时，自动恢复为 **正常**

这样可以有效避免人工维护状态带来的数据不一致问题。

### 3. 低库存预警联动
低库存预警不再依赖静态展示，而是基于最新产品数据动态生成：

- Alerts 页面自动展示低库存产品
- Dashboard 页面低库存预警数量自动更新
- 库存恢复正常后，预警信息自动移除

### 4. 出入库驱动库存变化
Transactions 页面已具备基础交易闭环能力：

- 新增入库记录
- 新增出库记录
- 出库库存不足自动拦截
- 每笔交易自动影响产品库存数量
- 最近出入库记录可在 Dashboard 中实时展示

### 5. 多页面数据联动
当前已经实现以下页面之间的数据联动：

- Products
- Transactions
- Dashboard
- Alerts

产品库存一旦因编辑或出入库发生变化，相关统计、预警和展示内容会自动重新计算并保持一致。

## 项目结构

以下为当前项目核心结构示意：

```bash
src/
├── pages/
│   ├── Products.jsx
│   ├── Transactions.jsx
│   ├── Dashboard.jsx
│   ├── Alerts.jsx
│   ├── Users.jsx
│   ├── Settings.jsx
│   └── Login.jsx
├── services/
│   └── productService.js
├── constants/
│   └── mockData.js
├── layouts/
│   └── MainLayout.jsx
├── components/
│   └── ...
└── App.jsx
```
结构说明
pages/：页面级模块
services/productService.js：产品、库存、交易记录等核心业务逻辑
constants/mockData.js：初始 mock 数据
layouts/：主布局结构
components/：通用组件与页面复用组件

可按实际代码结构继续调整。

技术栈
React
Vite
Tailwind CSS
项目特点
独立于旧库存系统，不影响现有系统使用
风格简洁、专业、克制，偏企业后台
使用 mock 数据，便于前端结构和 UI 演示
避免营销官网风格和明显 AI 拼装感
适合作为后续真实业务开发的前端基础
采用分阶段迭代方式推进，便于持续扩展与维护

## 当前阶段边界

当前项目仍采用 **前端内存 mock 数据** 方案，因此：

- 页面刷新后，新增记录和库存变化会恢复为初始 mock 数据
- 尚未接入真实后端接口
- 尚未接入数据库持久化存储
- 尚未实现权限管理、审批流、批量导入导出等企业级功能

这些属于当前阶段设计边界，不影响项目作为业务原型的展示价值。
