# 库存自动化管理系统 V2

一个基于 **React + Vite + Tailwind CSS** 构建的库存自动化管理系统前端原型项目，面向实验室耗材与库存管理场景。

本项目是对旧版库存系统的 **独立重做 V2 版本**，目标不是直接改造现有线上系统，而是在不影响旧系统使用的前提下，重新搭建一个结构更清晰、视觉更统一、便于后续扩展真实业务逻辑的新版前端基础。

---

## 项目定位

本项目的核心目标包括：

- 重设计库存系统 UI
- 优化后台布局与信息层级
- 提升系统的专业感、规范感与品牌感
- 以 mock 数据完成前端业务原型搭建
- 为后续接入真实后端、数据库与业务逻辑打下清晰基础

项目当前重点不是“快速接后端”，而是先把前端业务骨架、交互流程和多页面联动能力做扎实。

## 当前已实现能力

### 产品与库存管理
- **产品管理闭环**：新增、编辑、删除产品，支持搜索、分类筛选与分页联动
- **库存状态自动计算**：根据 `currentStock` 与 `minStock` 自动推导库存状态（正常/低库存），无需人工维护
- **低库存预警联动**：Alerts 页面动态展示低库存产品，Dashboard 预警数量自动更新
- **多页面数据同步**：Products、Dashboard、Alerts 基于统一数据源，库存变化后各页面自动刷新

### 交易与库存联动
- **出入库记录管理**：新增入库/出库交易，出库时自动进行库存不足校验
- **交易驱动库存变化**：每笔交易自动影响产品库存数量，库存变化自动触发状态重新计算
- **交易撤销与回滚**：支持撤销已完成交易，自动执行库存回滚并标记已撤销状态
- **最近交易记录展示**：Dashboard 实时展示最近出入库记录与状态

### 审计日志与操作追溯
- **审计日志底层系统**：自动记录产品增删改、出入库、交易撤销、系统重置等关键操作
- **Dashboard 最近操作记录**：首页展示最近 5 条操作日志，快速感知系统动态
- **AuditLog 独立页面**：完整日志列表查看，按时间倒序展示所有操作记录
- **基础筛选能力**：支持按操作类型筛选、按产品名称关键词搜索，提升日志查找效率

### 系统维护与开发调试
- **localStorage 本地持久化**：产品、交易、日志数据刷新后不丢失，适合单机演示与测试
- **Settings 重置本地测试数据**：提供安全可控的 UI 入口，支持一键重置所有本地数据到初始状态
- **SYSTEM_RESET 审计记录**：系统重置操作自动记录审计日志，保持操作可追溯性

## 第二阶段进展

### 第一轮：产品列表交互化
产品页面从静态展示升级为可交互原型，实现搜索、分类筛选、分页联动与操作按钮事件入口。

### 第二轮：产品增删改闭环
完成产品模块前端 mock 数据闭环，实现新增、编辑、删除产品的表单弹窗交互与数据变化后的列表联动。

### 第三轮：库存状态与预警联动
引入库存状态自动计算规则，打通 Products、Dashboard、Alerts 多页面数据同步，修复不同页面数据源不一致问题。

### 第四轮：出入库与库存同步
实现交易记录驱动库存变化，新增入库/出库记录表单，库存变化自动触发状态重新计算与页面联动。

### 第五轮：localStorage 本地持久化
为系统增加本地持久化能力，产品、交易、日志数据刷新后不丢失，从内存演示升级为可保留状态的原型。

### 第六轮：交易撤销与库存回滚
实现交易撤销能力，支持撤销已完成交易并自动执行库存回滚，增加安全校验与状态标记。

### 第七轮：审计日志底层能力
建立审计日志系统，自动记录产品增删改、出入库、交易撤销等关键操作，实现操作可追溯性基础。

### 第八轮：Dashboard 最近操作记录
在 Dashboard 首页新增最近操作记录模块，展示最近 5 条日志，让审计日志从底层记录升级为页面可感知。

### 第九轮：Settings 重置本地测试数据
为 `resetStorageData()` 提供页面级操作入口，增加二次确认与安全提示，补齐服务层能力与页面操作闭环。

### 第十轮：AuditLog 独立日志页面
新增 AuditLog 独立页面，提供完整日志列表查看能力，支持按时间倒序展示所有操作记录。

### 第十一轮：AuditLog 基础筛选能力
为 AuditLog 页面增加操作类型筛选与产品名称关键词搜索能力，让日志从“能看”升级为“能查”。

### 第十二轮：系统交互体验微调
对 AuditLog、Dashboard、Settings 等页面进行轻量体验优化，统一文案表达与视觉细节，提升完成感。

## 项目结构

以下为当前项目核心结构示意：

```bash
src/
├── pages/
│   ├── Products.jsx
│   ├── Transactions.jsx
│   ├── Dashboard.jsx
│   ├── Alerts.jsx
│   ├── Users.jsx
│   ├── Settings.jsx
│   └── Login.jsx
├── services/
│   └── productService.js
├── constants/
│   └── mockData.js
├── layouts/
│   └── MainLayout.jsx
├── components/
│   └── ...
└── App.jsx
```
结构说明
pages/：页面级模块
services/productService.js：产品、库存、交易记录等核心业务逻辑
constants/mockData.js：初始 mock 数据
layouts/：主布局结构
components/：通用组件与页面复用组件

可按实际代码结构继续调整。

技术栈
React
Vite
Tailwind CSS
项目特点
独立于旧库存系统，不影响现有系统使用
风格简洁、专业、克制，偏企业后台
使用 mock 数据，便于前端结构和 UI 演示
避免营销官网风格和明显 AI 拼装感
适合作为后续真实业务开发的前端基础
采用分阶段迭代方式推进，便于持续扩展与维护

## 当前阶段边界

当前项目仍采用 **前端内存 mock 数据** 方案，因此：

- 页面刷新后，新增记录和库存变化会恢复为初始 mock 数据
- 尚未接入真实后端接口
- 尚未接入数据库持久化存储
- 尚未实现权限管理、审批流、批量导入导出等企业级功能

这些属于当前阶段设计边界，不影响项目作为业务原型的展示价值。
