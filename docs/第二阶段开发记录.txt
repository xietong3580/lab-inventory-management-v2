# V3 第一轮开发记录
## 功能名称
Dashboard 报表增强（专业版起步）

## 本轮目标
在现有 Dashboard 基础上，增强管理概览与数据表达能力，让页面从“基础总览页”升级为更像现代企业后台的管理驾驶舱，同时保持轻量实现，不接入后端、不引入复杂依赖。

## 开发背景
第二阶段已完成：
- 产品与库存管理闭环
- 出入库与库存联动
- localStorage 本地持久化
- 审计日志系统
- Dashboard 最近操作记录
- Settings 重置本地测试数据入口
- AuditLog 独立页面
- AuditLog 基础筛选能力
- 第二阶段 README 与阶段总结整理

当前系统已具备较完整的业务链路，但 Dashboard 仍以基础信息总览为主，缺少更强的管理视角、趋势表达与概览统计。因此第三阶段第一轮的重点是增强 Dashboard 的报表表达能力。

## 本轮预期实现
1. 增强 Dashboard 核心 KPI 卡片
2. 增加近 7 日交易趋势可视化
3. 增加低库存概览增强模块
4. 增加近 7 日审计记录概览统计
5. 优化 Dashboard 信息层级，使其更像管理驾驶舱

## 本轮设计原则
- 不新增底层业务逻辑
- 不接入后端或数据库
- 不引入复杂图表库
- 优先使用纯 CSS / HTML 的轻量可视化方式
- 保持企业后台风格统一
- 强调清晰、结构化、可读性

## 本轮建议展示模块
### 1. 增强 KPI 卡片
建议重点展示：
- 产品总数
- 正常库存产品数
- 低库存产品数
- 近 7 日交易
- 近 7 日审计记录

### 2. 交易趋势模块
- 基于现有交易记录计算近 7 日入库 / 出库趋势
- 使用轻量条形图或柱状可视化方式展示

### 3. 低库存概览增强
- 低库存数量
- 低库存占比
- 最需关注的前 3 个产品

### 4. 审计日志概览统计
- 对近 7 日审计记录按操作类型进行概览统计
- 使用清晰的条形统计或轻量分类统计表达

## 本轮暂不包含
- 后端 API
- 数据库
- 权限体系
- 导出能力
- 高级图表联动
- 高级筛选面板
- 报表下载

## 本轮价值
本轮将 Dashboard 从“基础概览页”升级为“管理驾驶舱”，提升系统的专业感、管理感与展示价值，也为第三阶段后续更完整的报表与数据层升级打下基础。
