# Project Overview

本项目是“库存自动化管理系统 V2”，是一个独立于现有旧系统之外的全新版本。

目标：
- 不影响当前已上线使用的旧库存系统
- 先完成新版前端界面与后台 UI 重设计
- 提升系统界面专业度，减少传统后台土味感和 AI 拼装感
- 后续再逐步接入真实业务逻辑和数据

当前阶段：
- 第一阶段只做前端页面和交互框架
- 使用 mock 数据演示
- 暂不接入真实数据库、真实登录鉴权和复杂业务逻辑

# Tech Stack

- React
- Vite
- Tailwind CSS

# Project Rules

1. 这是一个全新独立项目，不能影响旧系统。
2. 第一阶段只做前端页面，不做真实后端接入。
3. 暂时不做真实数据库、真实登录鉴权、权限系统、报表导出、备份恢复等复杂功能。
4. 页面开发优先使用 mock 数据。
5. 一次只完成一个页面，不要一口气生成整个系统所有页面。
6. 每次改动后，都要明确说明：
   - 改了哪些文件
   - 为什么这样改
   - 下一步我需要在 VS Code 终端执行什么命令
7. 如果发现需求需要调整，先解释原因，再改动。

# UI Style Rules

整体风格要求：
- 现代企业后台风格
- 简洁、专业、克制
- 少 AI 味
- 不要科技炫光风
- 不要蓝紫渐变大面积使用
- 不要过度卡片化
- 不要厚重阴影
- 不要为了“高级感”堆太多装饰

界面规范：
- 以蓝灰、中性色为主
- 强调清晰层级、留白、对齐和信息密度控制
- 登录页、仪表盘、产品页、出入库页、预警页都要保持统一设计语言
- 图标风格统一
- 按钮样式层级统一
- 表格、表单、卡片、侧边栏风格统一

# Page Scope (Phase 1)

第一阶段优先页面：
1. 登录页
2. 仪表盘首页
3. 产品管理页
4. 出入库记录页
5. 低库存预警页

# Not In Scope (Phase 1)

第一阶段暂不做：
- 真实数据库接入
- 真实登录鉴权
- 用户权限系统
- Excel 导入导出
- 报表生成
- 自动备份恢复
- 多仓库支持

# Working Style

请按小步快跑方式协助开发：
- 先做结构
- 再做样式
- 再做页面细化
- 再做页面之间统一
- 最后再考虑交互完善

如果当前任务只是登录页，就不要提前扩展到其他所有页面。
优先保证当前页面完成度，而不是贪多。
